================================================================================
              RATE LIMIT PROTECTION (CRITICAL FEATURE)
================================================================================

Every single API call passes through: RateLimit.canCall(apiName)

Budget Enforcement Table:
┌─────────────────┬──────────────┬─────────────┬──────────────┐
│ API Name        │ Monthly      │ Daily       │ Hourly       │
├─────────────────┼──────────────┼─────────────┼──────────────┤
│ Exa Search      │ 1,000        │ 24 max      │ 1            │
│ Crypto.com      │ Unlimited    │ Unlimited   │ 60           │
│ Deribit         │ Unlimited    │ Unlimited   │ 60           │
│ Fear & Greed    │ Unlimited    │ 1           │ 1            │
│ BigData.com     │ Unlimited    │ 2           │ 1            │
│ Kronos AI       │ Unlimited    │ 1           │ 1            │
│ Binance OHLCV   │ Unlimited    │ Unlimited   │ 60           │
└─────────────────┴──────────────┴─────────────┴──────────────┘

Exa Search Safety Analysis:
├─ Official limit: 1,000 searches/month
├─ Per day: 1,000 ÷ 30 = 33.3 searches/day
├─ Our usage: 1 per hour × 24 = 24 searches/day
├─ Buffer: 33.3 - 24 = 9.3 unused searches/day
├─ Monthly buffer: 9.3 × 30 = 279 unused searches
└─ Safety margin: 9.3 ÷ 33.3 = 27% (EXCELLENT)

Storage: localStorage tracked by hour/day/month windows
Fallback: When limit exceeded, cached data is served
Display: Dashboard shows live usage in rate limit panel

================================================================================
                        FREQUENTLY ASKED QUESTIONS
================================================================================

Q: Do I need an Exa API key to use the dashboard?
A: No! The dashboard works perfectly without it. It falls back to cached 
   static news on GitHub Pages. Local testing also works without it.

Q: Where does my Exa API key go?
A: Into src/config.js locally (in .gitignore). Never committed to GitHub. 
   Your key stays private.

Q: Will my API key be exposed?
A: No. The file is in .gitignore so Git won't track it. Never push it 
   manually. On GitHub Pages, the file won't exist - the dashboard uses cache.

Q: How often do the APIs get called?
A: Price: every 60s (cheap). Full refresh: every 60min (respects limits). 
   Kronos: 1/day at 09:00 IST. News: 1/hour. Options: 1/10min. 
   All enforced by RateLimit.js.

Q: Will I exceed the Exa rate limit?
A: No. We use 24/day max with a 27% safety buffer (33/day official limit). 
   Even with automatic refreshes, it's impossible to exceed.

Q: How does the cache system work?
A: localStorage stores API responses with TTL (time-to-live). When an API 
   limit is exceeded or network is slow, cached data is served instead.

Q: What happens if an API is down?
A: Dashboard falls back to cached data from the last successful fetch. 
   It keeps working with stale data until the API recovers.

Q: Can I deploy to Vercel or Render instead?
A: Yes, but GitHub Pages is simpler for this use case. It's free, fast, 
   and automatic - just push code, it deploys instantly.

Q: How do I update the dashboard?
A: Make changes locally, commit with git, and push: git push origin main. 
   GitHub auto-deploys within 30 seconds.

Q: How do I see deployment status?
A: Go to: https://github.com/YOUR_USERNAME/btc-dashboard/actions
   Each commit triggers a deployment workflow you can monitor.

Q: Does it work offline?
A: Partially. Cached data will display, but fresh API data won't load. 
   The cache persists in localStorage even after closing the browser.
